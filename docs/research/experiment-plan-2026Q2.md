# boby-grow-weapp 增长实验方案（2026Q2）

- 主目标：验证导航收敛与闭环机制对留存/活跃的提升
- 配套报告：`docs/research/growth-optimization-report-2026-03-27.md`

## 1. 指标口径

1. D7 家庭留存率
- 分子：首日激活后第7天仍有关键行为的家庭
- 分母：首日激活家庭数

2. 周活跃家庭（WAF）
- 近7天至少发生2次关键行为的家庭数

3. 任务闭环率
- `已提交且审核完成任务 / 应完成任务`

4. 审核处理时长
- 从提交到审核完成的中位时长（小时）

## 2. 实验设计

- 实验对象：父母端导航+审核聚合+奖励闭环改造
- 分组方式：灰度分流（10% -> 30% -> 100%）
- 对照方式：新旧导航并行观察
- 观察周期：每阶段至少7天，总计4-6周

## 3. 埋点方案

必需事件：
- `onboarding_step_completed`
- `task_cycle_closed`
- `audit_turnaround_time`
- `reward_loop_closed`

统一字段：
- `role`
- `childId`
- `familyId`
- `ts`
- `source`

## 4. 验收阈值（默认）

- D7 家庭留存：+15%（相对）
- 周活跃家庭：+12%（相对）
- 任务闭环率：+18%（相对）
- 审核处理时长：-25%（相对）

> 上述阈值在实验前以最近2周基线复核后锁定。

## 5. 风险与回滚

1. 若导航改造导致首日激活下滑 >5%，立即暂停扩量。
2. 若审核消息误触发导致用户投诉增加，回滚红点策略。
3. 若埋点缺失率 >2%，该阶段数据判定无效，需补采重跑。

## 6. 输出节奏

- 每周输出一版实验周报（留存、活跃、闭环、时长）
- 每阶段结束输出 Go/No-Go 结论
