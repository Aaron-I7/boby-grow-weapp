# 家庭版儿童积分管理微信小程序 产品需求文档（PRD）

**文档核心信息**：适配家庭场景、儿童教育友好型积分管理小程序，核心解决低龄儿童无微信、家长精细化管控、积分正向激励三大痛点，预留MCP接口支持豆包等工具查询，全程极简操作、无复杂流程。

**文档版本**：V1\.0 正式版 \| **适用人群**：产品、开发、UI设计 \| **核心设计原则**：教育正向激励、无微信儿童适配、家长全权管控、操作极简

# 一、项目整体需求说明

## 1\.1 项目背景与核心目标

针对家庭育儿场景，打造专属儿童积分管理工具，摒弃金钱奖励，用积分可视化正向强化孩子的好习惯、家务意识与学习主动性，兼顾**低龄儿童无微信使用**和**大龄儿童有微信自主查看**双场景；家长作为管理员全权定制规则，孩子仅参与规则确认、任务查看与积分查询，同时预留MCP接口，支持孩子通过豆包等AI工具自主查询积分、任务详情，解决无微信儿童的使用缺口，最终实现育儿规则透明化、孩子行为正向引导、家庭协同管理。

## 1\.2 核心业务需求（用户提报\+补充优化）

1. **管理员（家长）权限**：自定义积分规则、任务体系，管理孩子信息，绑定多家长/多孩子，查看积分与任务数据，审核奖励兑换，全程管控积分发放与扣除。

2. **儿童端需求**：无需强制绑定微信即可参与，可完成规则确认；有微信的儿童可绑定小程序自主查看，无微信儿童通过MCP接口（豆包）查询个人积分、可做任务、积分明细。

3. **基础管理需求**：任务全生命周期管理、个人信息管理、儿童档案管理、亲属关系绑定、多家长协同管理。

4. **扩展需求**：预留MCP接口，仅开放查询权限，保障数据安全；教育友好型设计，少扣分、多正向激励，奖励以陪伴类、特权类为主，杜绝功利化奖励。

5. **补充刚需**：数据可追溯、操作留痕，避免家长误操作；适配低龄儿童，界面大字、卡通化、少文字，语音引导可选。

## 1\.3 非功能需求

- **易用性**：家长端操作步骤≤3步/核心功能，儿童端无复杂操作，老人也可快速上手家长端。

- **安全性**：儿童数据仅家庭内部可见，MCP查询需专属核验码，禁止积分操作、奖励兑换等敏感权限开放；儿童隐私严格保护，不收集敏感信息。

- **兼容性**：适配微信小程序全版本，支持手机、平板端打开，页面自适应不同尺寸。

- **性能**：页面加载≤2秒，核心操作（加分、规则保存）无延迟，支持离线查看基础数据。

# 二、角色与权限划分（核心：不区分儿童有无微信）

本项目**绑定环节不强制区分儿童是否有微信**，仅以「儿童档案」为核心数据载体，微信账号仅为可选入口，MCP为补充查询入口，彻底解决低龄儿童无微信的使用痛点，角色权限划分如下：

|角色|身份定义|核心权限|使用入口|
|---|---|---|---|
|**主家长（超级管理员）**|创建家庭档案、初始管理员，通常为父母一方|全功能权限：定制规则、管理孩子/协管员、发放积分、审核奖励、数据查看、MCP配置、解绑操作|微信小程序家长端|
|**协管家长（普通管理员）**|被主家长邀请的亲属，如另一方父母、祖辈|查看孩子数据、标记任务完成、加分、查看奖励申请，无删除孩子、修改核心规则权限|微信小程序家长端|
|**儿童（核心用户）**|被家长创建档案的孩子，无账号属性，仅关联档案|查看任务/积分、确认规则、申请奖励（可选）、查询积分明细；无积分操作、规则修改权限|1\. 无微信：MCP接口（豆包）\+ 线下确认2\. 有微信：可选绑定小程序儿童端|

**核心逻辑**：儿童无独立微信账号，仅需家长创建**儿童专属档案**，生成唯一档案ID\+6位查询核验码，即可通过MCP查询；有微信的儿童可扫码绑定档案，实现小程序端自主查看，数据完全互通，不影响核心功能。

# 三、全模块功能详细设计

整体分为**家长管理端**、**儿童端（微信小程序可选）**、**MCP查询通道**三大模块，核心功能拆解如下：

## 3\.1 家长管理端（核心模块，微信小程序）

### 3\.1\.1 家庭与人员管理模块

- **儿童档案管理**：主家长新增孩子档案，填写姓名、昵称、年龄、卡通头像（系统内置可选，禁止自定义上传），生成唯一档案ID\+6位查询核验码（用于MCP查询）；支持编辑档案、归档/删除闲置档案，多孩子可切换查看。

- **多家长协同绑定**：主家长生成邀请码/二维码，邀请其他微信用户成为协管家长，绑定后数据同步；支持解除协管权限，权限分级管控。

- **个人信息管理**：修改家长昵称、头像，设置操作提醒开关，查看个人操作日志。

### 3\.1\.2 积分规则定制模块

- **规则分类自定义**：预设好习惯、学习任务、家务劳动、品德行为四大分类，家长可新增/编辑分类，自定义每个行为的积分值（仅正向加分，默认关闭扣分功能，需手动开启且仅允许少量扣分）。

- **规则限制设置**：设置每日积分上限、单任务每日可完成次数，避免刷分；规则启用/禁用切换，适配周末、假期等场景。

- **规则确认推送**：新规则/规则修改后，推送至儿童端，需儿童确认后生效（无微信儿童线下确认，家长标记；有微信儿童小程序端点击确认）。

### 3\.1\.3 任务管理模块

- **任务创建**：基于规则生成日常固定任务，也可创建临时任务；设置任务周期（每日/每周/单次）、任务说明、对应积分。

- **任务执行管控**：查看孩子今日/本周任务清单，手动标记任务完成，一键发放积分；支持撤销误操作，备注操作原因，全程留痕。

- **任务申请审核**：儿童自主提交的任务完成申请（拍照/文字），家长审核通过后加分，驳回可备注原因。

### 3\.1\.4 积分与奖励管理模块

- **积分管理**：查看孩子实时积分、总积分、积分明细（时间、任务、操作人、积分变动）；支持手动调整积分（仅特殊情况，需备注）。

- **奖励商城定制**：家长自定义奖励清单，分陪伴类（亲子阅读、公园游玩）、特权类（选晚餐、晚睡10分钟）、实物类（小文具、玩具），设置所需积分、兑换上限。

- **奖励兑换审核**：查看孩子奖励兑换申请，审核通过后扣除对应积分，标记奖励已发放；拒绝申请需说明原因。

### 3\.1\.5 数据统计与MCP配置模块

- **数据统计**：生成孩子周/月积分趋势图、高频完成任务统计，可视化展示成长情况，支持导出数据记录。

- **MCP接口配置**：查看儿童专属查询核验码，配置接口基础权限（仅开放查询，关闭操作权限），设置查询话术模板，适配豆包等AI工具。

## 3\.2 儿童端（微信小程序，可选绑定）

仅为有微信的大龄儿童设计，极简轻量化，无任何操作权限，纯查看与确认：

- **档案绑定**：扫描家长提供的二维码，一键绑定儿童档案，支持解绑。

- **规则确认**：接收家长新规则推送，卡通弹窗展示，点击“我同意”完成确认，同步生效。

- **任务查看**：展示今日可完成任务、已完成任务，带卡通图标和积分说明。

- **积分查询**：展示当前积分、等级（卡通勋章）、积分明细。

- **奖励申请**：查看可兑换奖励，提交兑换申请，查看申请进度。

## 3\.3 MCP查询通道（核心，适配无微信儿童）

预留标准接口，对接豆包等AI工具，**仅开放查询权限，无任何操作权限**，保障数据安全：

- **身份核验**：儿童输入6位查询核验码，验证通过后查询个人数据，核验失败禁止访问。

- **核心查询功能**：


    1. 查询当前积分、总积分、当前等级

    2. 查询今日可完成任务及对应积分

    3. 查询近3日积分明细、任务完成情况

    4. 查询可兑换的奖励清单及所需积分

- **话术适配**：返回儿童化口语话术，无专业术语，简洁易懂，适配低龄儿童理解能力。

# 五、核心业务流程

### 5\.1 儿童档案创建流程

主家长进入小程序 → 新增孩子档案（填信息）→ 生成档案ID\+查询核验码 → 完成绑定 → 邀请协管家长（可选）→ 儿童可通过MCP查询（有微信可扫码绑定小程序）

### 5\.2 规则制定与生效流程

家长定制积分规则/任务 → 设置积分与次数 → 推送规则给儿童 → 儿童确认（线下/小程序）→ 家长标记确认完成 → 规则正式生效 → 任务自动同步至任务清单

### 5\.3 积分获取与MCP查询流程

儿童完成任务 → 家长标记完成/审核通过 → 发放积分 → 积分实时更新 → 儿童通过豆包输入核验码 → 核验通过 → 查询积分/任务详情

# 六、数据安全与隐私说明

- 儿童数据仅绑定家庭档案，仅授权家长查看，不对外共享，严格保护儿童隐私

- MCP接口仅开放查询权限，禁止任何积分操作、奖励申请等敏感操作，核验码6位纯数字，防止暴力破解

- 家长操作全程留痕，积分变动可追溯，支持数据备份与导出，避免数据丢失

- 不收集儿童身份证、住址等敏感信息，仅记录姓名、年龄等基础档案信息
